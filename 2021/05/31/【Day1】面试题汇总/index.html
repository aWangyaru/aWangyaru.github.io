1, 声明 docType 的作用？

doctype是document.type的缩写（文档类型），声明文档的解析类型，避免浏览器的怪异模式。

浏览器解析CSS有两种模式：标准模式、怪异模式

标准模式：浏览器按W3C标准解析执行代码

怪异模式：使用浏览器自己的方式解析执行代码，因为不同浏览器的解析执行不一样，所以称为怪异模式。

两者区别：

1，盒模型: css盒模型本质上就是一个盒子，它包括：外边距(margin)、边框(border)、内边距(padding)、实际内容(content)四个属性。

W3C盒模型(标准盒模型)：

怪异盒模型(IE盒模型)：

标准模型和IE模型的区别

标准盒模型：盒子总宽度/高度 = width/height + padding + border + margin。（ 即 width/height 只是内容高度，不包含 padding 和 border 值 ）

IE盒子模型：盒子总宽度/高度 = width/height + margin = (内容区宽度/高度 + padding + border) + margin。（ 即 width/height 包含了 padding 和 border 值 ）

css 如何设置这两种模式

标准：box-sizing: content-box; ( 浏览器默认设置 )
IE： box-sizing: border-box;

2，position 属性及其作用

常见的属性：relative、absolute、fixed、static

1, static 默认值，始终处于页面流基于的位置

2，relative 相对定位 ，如果设置了relative，那么他的偏移的top/bottom/left/right的值都是以他原来的位置为基准偏移。relative移动后的元素在原来的位置扔占据空间

3，absolute 绝对定位，如果他的父容器设置了position属性，且值为 relative 或者 absolute，那么就会依据父容器进行偏移。如果父元素/父父元素…没有设置position属性，那么偏移就是依body为依据。absolute的元素在标准流中不占位置

4, fixed 固定定位，相对于浏览器窗口的指定坐标，不论窗口滚动与否，元素都会留在那个位置，始终以 body 为依据，

3，for…of、for…in区别

For..in是ES5标准，遍历的是Key，可遍历对象、数组或字符串的key。

for-of是ES6标准，遍历的是value，可遍历数组或字符串的value。

for-in会遍历数组所有的可枚举属性，包括原型。原型方法method和name属性都会被遍历出来，通常需要配合hasOwnProperty()方法判断某个属性是否该对象的实例属性，来将原型对象从循环中剔除。

for (var key in myObject) {
　　if（myObject.hasOwnProperty(key)){
　　　　console.log(key);
　　}
}

for in循环遍历的是数组的键值(索引)，而for of循环遍历的是数组的值。

1, 区别1 - for..of无法循环遍历对象

let arr = {
  a: 1,
  b: 2,
  c: 3
}
for (const i in arr) {
	console.log(i)
}
console.log('----------')
for (const i of arr) {
	console.log(i)
}

2, 区别2 - 遍历输出结果不同

let arr = ['a', 'b', 'c']
for (const i in arr) {
	console.log(i)
}
console.log('----------')
for (const i of arr) {
	console.log(i)
}

3, 区别3 - for..in可遍历自定义属性，for..of不可以

let arr = ['a', 'b', 'c']
arr.name = 'QQQ'
for (const i in arr) {
	console.log(i)
}
console.log('----------')
for (const i of arr) {
	console.log(i)
}
